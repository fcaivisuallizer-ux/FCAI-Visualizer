# AlgoViz: Handover & Full Implementation Roadmap

## 🤖 AI Handover Notes (READ FIRST)
This project is a migration of a legacy "AlgoViz" tool to **React 18 + Vite**. 
The architecture follows a strict **Separation of Concerns**:

1. **Generators (`/src/algorithms/[category]/...Generators.js`)**: 
   - Pure JS functions that take input data (e.g., an array) and return a `steps` array.
   - Each `step` object MUST contain: `arr` (state snapshot), `msg` (string), `codeLine` (int), and highlight metadata (`compare`, `swap`, `sorted`).
   - *Why?* This keeps algorithm logic decoupled from React's rendering cycle.

2. **Visualizers (`/src/algorithms/[category]/...Visualizer.js`)**:
   - These are **not** React components. They are helper classes/functions that handle Canvas or DOM-based drawing based on the `step` data.
   - They receive a `context` (CanvasRef) and a `step` object.

3. **Pages (`/src/pages/...`)**:
   - Manage React state (current step index, speed, playing/paused).
   - Sync state with URL params (e.g., `?algo=bubble&speed=50`).
   - Use `useEffect` to trigger the Visualizer whenever the `currentStep` changes.

## ✅ What has been done (Detailed)
- **Infrastructure**: Complete. Uses `lucide-react` for icons and has a custom `ThemeContext` for Dark/Light mode.
- **Sorting Module**: Fully migrated. 
  - Added support for 8 algorithms.
  - Implemented 5 different "Views": **Bar Chart**, **Cell Grid**, **Merge Tree** (Recursive), **Quick Partition** (Recursive), and **Radix Buckets**.
  - **Dynamic Code Highlighting**: Synchronized with 5 languages (JS, Python, C++, Java, C#).
- **Trees Module**: AVL Tree visualization is functional.
  - Uses a coordinate-based recursive drawing system for nodes.
- **Mobile Responsiveness**: The sidebar collapses on mobile, and control panels wrap or scroll.

## 🎯 Target Goal
The final version should be a **Single Page Application (SPA)** where a student can:
1. Select any algorithm.
2. See the code in their preferred language.
3. Step through the visualization while reading the explanation.
4. Export the current state or share it via a URL link.

## ⚠️ Known Issues / Gotchas for the next AI:
- **Canvas Scaling**: If the window resizes while an animation is playing, the canvas might blur. The `useResizeObserver` hook should be used to fix this.
- **Generator Performance**: For very large arrays, generators can consume memory. Keep default array sizes < 50.
- **Legacy Logic**: Some parts of the old `BST.js` still use `window.alert`. These MUST be replaced with a custom `toast` or `Notification` component.
